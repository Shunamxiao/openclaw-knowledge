# 📁 知识库目录结构说明

本仓库用于存储 OpenClaw 运行过程中的经验总结和配置备份。

## 📂 目录结构

```
knowledge_base/
├── README.md                 # 本说明文件
├── 2026-02-03.md            # 每日经验总结（按日期命名）
├── 2026-02-04.md            # 每日经验总结
├── cron-backups/            # ⏰ 定时任务备份目录
│   ├── README.md           # 定时任务备份说明
│   ├── latest.md           # 最新完整备份（ always up-to-date ）
│   └── 2026-02-04_1043.md  # 历史备份（带时间戳）
└── [其他技能文档].md       # 各种技能使用指南
```

## 📖 各目录用途

### 1. 根目录 (/) - 每日经验总结
- **命名规则**: `YYYY-MM-DD.md`
- **内容**: 当天工作的重要经验教训、解决方案、配置技巧
- **更新方式**: 每天 00:00 自动推送

### 2. cron-backups/ - 定时任务备份
- **用途**: 存放所有定时任务的配置备份
- **更新时机**: 
  - 每次添加/修改/删除定时任务时
  - 服务重启后发现任务丢失时
- **使用方法**: 查看 `latest.md` 获取最新配置

### 3. 其他技能文档
- 各种技能的使用指南和配置说明
- 命名建议: `技能名称-guide.md` 或 `主题名称.md`

## 🔄 自动推送机制

**每天 00:00** 自动执行：
1. 检查当日会话内容
2. 提取重要经验教训
3. 生成/更新 `YYYY-MM-DD.md`
4. 推送到 GitHub

## 📝 手动添加内容

### 方式一：通过对话
告诉我："记录到知识库：xxx"

### 方式二：直接编辑
在本地 `skills/knowledge_base/` 目录创建文件，下次定时任务会自动推送。

## 🔍 快速查找

- **找定时任务配置**: 查看 `cron-backups/latest.md`
- **找某天的经验**: 查看 `YYYY-MM-DD.md`
- **找技能指南**: 查看根目录下的其他 `.md` 文件
